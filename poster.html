Causal Inference for Testing Computational Models

Testing Models is Hard
  • Long runtimes
  • Computationally expensive
  • Vast, uncertain input space
  • Expected outputs unknown
  • Nondeterministic
Testing is a Causal Activity

Causal tests specify the expected effect of changing the input on the output. This is very similar to causal questions in causal inference, which addresses many of the challenges we face.

Aims and Objectives
  • Use causal reasoning to test models
  • Use CI to maximise data efficiency
  • Automatically generate causal tests
  • Measure test adequacy causally
  • Use causal reasoning to locate faults

Test Case Generation
Which input values do we want to test?

Users want to test abstract properties like "When we double the intensity, num_shapes_unit should quadruple".

CI needs concrete properties like "When we change the intensity from I=i to I=i', num_shapes_unit should quadruple".

We need a way of choosing which treatment values to use to get a good coverage of the input space using as few tests as possible. Latin Hypercube Sampling seems like a promising technique.

Test Adequacy
How do we know when we've done enough tests?

We need a way to measure our coverage of modelling scenarios so we know when we've done enough tests.

We can use the causal structure as a basis for this: If every edge is sufficiently covered, then the model is sufficiently tested.

What does it mean to "cover" an edge? If we generate test inputs according to a statistical distribution, we can use KS tests to determine whether our tests form a statistically significant sample.

Causal Testing Framework
  • Applies techniques from causal inference to software testing
  • Users can write causal tests at a high level, without needing to understand CI
  • Works "out of the box", but is highly customisable
  • Implemented in Python, but can work with any model
